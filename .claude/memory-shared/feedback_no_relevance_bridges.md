name: feedback_no_relevance_bridges
description: When writing any prose output (emails, cover letters, code comments, PR descriptions) — avoid connector sentences that explain how the content relates to something else
metadata:
  type: feedback
---
Do not write explanatory bridge sentences that tell the reader how one thing relates to or maps onto another. Lines like "Since X pairs A with B, that's very familiar territory," "This maps cleanly to Y features," or "this mirrors X's behavior" read as AI-generated filler.
**Why:** Readers (recruiters, colleagues, code reviewers) are capable of drawing the parallel themselves. Overexplaining the connection is a tell that the text was machine-written, and it sounds like it is trying too hard.
**How to apply:** State facts directly in plain prose or bullets. Let the reader make the connection. If a link really must be surfaced, do it with one crisp noun phrase rather than a full "this-is-why-it-matters" sentence. This applies to all output types: outreach emails, application copy, code comments, docstrings, and PR descriptions.
